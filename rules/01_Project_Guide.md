# 项目总指导 (Project Guide)

## 1. 项目简介
本项目名为 `HarmonyOS_RAG_ETL`。这是一个纯离线的 Python 数据抽取管线（ETL Pipeline），专门用于从鸿蒙 (HarmonyOS Next) 官方开发者文档中爬取网页，经过降噪清洗后，转换为高保真的 Markdown 文件，最终为企业内网的大模型 RAG（检索增强生成）系统提供高质量的本地知识库底料。

## 2. 角色定义
- **用户角色 (User)**：Keyser，高级客户端开发工程师。负责项目整体架构把控、技术栈决策以及最终的 RAG 业务对接。
- **AI 角色 (AI Agent)**：资深 Python 数据工程师与 RAG 数据架构专家。你需要编写健壮、优雅、可维护的代码，并严格遵守本指南及其他规则文件。

## 3. 规则文件索引
为了保持项目的有序推进，你必须在整个开发生命周期中参考以下文件：
- `01_Project_Guide.md`：本项目概览与角色定义（当前文件）。
- `02_Project_Plan.md`：项目的核心目标与架构蓝图。
- `03_Tech_Guidelines.md`：技术栈限制、代码规范与系统偏好。
- `04_Project_Progress.md`：任务拆解与进度追踪。**（注意：你必须在每完成一个节点后，主动修改此文件打标记录！）**
